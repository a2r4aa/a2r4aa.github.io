Dkms Error when using a higher kernel

2020-05-06
Article Directory

This article was last updated on <span id="expire-date"></span> days ago, the information described in the article may be outdated.

After I upgraded my kernel to 5.5 version on Debain 10, the error happened when I tried to install WireGuard.

Error! The dkms.conf for this module includes a BUILD_EXCLUSIVE directive which
does not match this kernel/arch. This indicates that it should not be built.
Skipped.

How to solve this error?
I just deleted the older boot files in folder /boot.

Then I installed dkms apt install dkms. After I rebooted it, the error disappered.
